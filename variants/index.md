# Landing Page Variations

Five reviewable homepage directions for Parity. Each keeps the product message centered on matched coverage: how much a single test covers the file it belongs to.

<div class="variant-index">
  <a href="/variants/ledger">
    <span>01</span>
    <strong>Ledger</strong>
    <em>Stripe-like executive proof, with matched coverage as a source/test ledger.</em>
  </a>
  <a href="/variants/developer">
    <span>02</span>
    <strong>Developer</strong>
    <em>Command-first quickstart for engineers who want to see the config and output immediately.</em>
  </a>
  <a href="/variants/manual">
    <span>03</span>
    <strong>Manual</strong>
    <em>Documentation-first operating manual with a clear install, configure, enforce path.</em>
  </a>
  <a href="/variants/lab">
    <span>04</span>
    <strong>Lab</strong>
    <em>Sample-app showcase built around PHPUnit, Pest, Jest, Mocha, Vitest, and Rust.</em>
  </a>
  <a href="/variants/ci">
    <span>05</span>
    <strong>CI Room</strong>
    <em>Release-gate narrative focused on the exact failure a reviewer needs to fix.</em>
  </a>
</div>
